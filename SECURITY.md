# Security

## Supported versions

Security fixes are applied on the default branch (`main`). This project does not maintain separate LTS release branches.

## Reporting a vulnerability

**Do not open a public GitHub issue** for undisclosed security problems.

Please report privately using one of these options:

1. **GitHub:** On the repository page, open **Security → Advisories → Report a vulnerability** (available when repository owners enable private vulnerability reporting under repository Settings).
2. **Maintainer contact:** If maintainers list a security contact email or form in the README or org profile, use that channel.

Your report should include:

- What is affected and the potential impact
- Steps to reproduce or a minimal proof of concept, if it can be shared safely
- The affected area (e.g. a specific API route or dependency), if known

Maintainers should acknowledge receipt when possible and coordinate a fix and public disclosure timeline.

## Scope

**In scope:** Code and configuration in this repository, and deployment behavior described in the README.

**Out of scope:** Vulnerabilities in third-party services (Supabase, OpenAI, hosting providers); report those through their own programs. Social engineering and resource exhaustion against production systems without prior agreement are out of scope.
